# MVP 团队执行 TODO

## 1. 目标

- 在当前代码骨架基础上完成首版可用发布。
- 以“可演示、可回归、可上线”为最低标准。

## 2. 角色与任务分配

### 产品负责人

- 完成 MVP 功能清单冻结。
- 定义验收标准和演示脚本。

### 后端负责人（CoreApi）

- 完成会话加入与消息发送协议文档。
- 增加消息幂等去重（`messageId`）处理。
- 增加错误码与统一异常结构。

### 前端负责人（AdminPortal）

- 完成消息输入交互（回车发送、禁用态）。
- 增加连接状态与重连提示。
- 完成基础主题变量整理。

### 测试负责人

- 覆盖核心链路：连接、入会话、发消息、断线重连。
- 输出冒烟测试脚本与缺陷清单。

### SRE / 发布负责人

- 准备环境变量模板与部署说明。
- 准备健康检查与最小监控项。

## 3. 里程碑

- D1：接口与页面联通。
- D2：冒烟测试通过。
- D3：预发布演示通过。
- D4：MVP 正式发布。
