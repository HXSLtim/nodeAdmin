# MVP 发布清单

## 1. 发布前（必须完成）

- [ ] `CoreApi` 能通过构建。
- [ ] `AdminPortal` 能通过构建。
- [ ] `health` 接口可用（`GET /health`）。
- [ ] WebSocket 可完成入会话与发送消息链路。
- [ ] 环境变量模板已准备并审阅。
- [ ] 回滚方案已确认。

## 2. 冒烟验证（发布窗口）

- [ ] 打开前端页面，连接状态为 `connected`。
- [ ] 发送消息成功并实时回显。
- [ ] 断开网络后恢复连接可继续发送。
- [ ] 多浏览器窗口同会话可互相收到消息。

## 3. 发布后（30 分钟内）

- [ ] 观察服务健康检查与错误日志。
- [ ] 观察消息投递与连接数趋势。
- [ ] 核对关键链路无高优先级告警。

## 4. 失败回滚条件

- 连续 5 分钟不可用。
- 消息投递失败率超过阈值。
- 核心页面无法连接 WebSocket。

## 5. 责任人

- 发布指挥：项目负责人。
- 技术执行：后端负责人 + 前端负责人。
- 质量确认：测试负责人。
- 稳定性确认：SRE。

## 6. 最近更新时间

- 2026-02-28
